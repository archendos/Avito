Задание 2, Вариант 1
Тест кейсы для проверки API микросервиса:
Тест-кейс: Успешное создание объявления.

Шаги:
Отправить запрос POST для создания объявления с валидными данными.
Проверить, что возвращается статус 201.
Убедиться, что объявление сохранено с корректными данными.
Ожидаемый результат: Объявление успешно создано, возвращены данные нового объявления.
Тест-кейс: Получение объявления по идентификатору.

Шаги:
Отправить запрос GET с существующим идентификатором объявления.
Проверить, что возвращается статус 200 и корректные данные объявления.
Ожидаемый результат: Возвращаются корректные данные объявления.
Тест-кейс: Получение всех объявлений по идентификатору продавца.

Шаги:
Отправить запрос GET с идентификатором продавца.
Проверить, что возвращается статус 200 и список всех объявлений данного продавца.
Ожидаемый результат: Возвращается список объявлений, связанных с указанным продавцом.
Тест-кейс: Ошибка при создании объявления с некорректными данными.

Шаги:
Отправить запрос POST для создания объявления с некорректными данными (например, пустые поля).
Проверить, что возвращается статус 400.
Ожидаемый результат: Возвращается ошибка с описанием некорректных данных.
